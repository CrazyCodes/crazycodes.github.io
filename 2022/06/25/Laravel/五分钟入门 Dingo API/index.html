概述

Dingo API帮助您轻松快速地构建自己的API。虽然这个方案的目标是尽可能保持灵活性，但它仍然不能涵盖所有情况并解决所有问题。

配置

在.env文件中配置你的Dingo API

  • API_STANDARDS_TREE
  • API_SUBTYPE
  • API_PREFIX
  • API_VERSION
  • API_NAME
  • API_CONDITIONAL_REQUEST
  • API_STRICT
  • API_DEBUG
  • API_DEFAULT_FORMAT

API_STANDARDS_TREE

Standards Tree 标准树

  • 未注册的树（x）主要表示本地和私有环境
  • 私有树（prs）主要表示没有商业发布的项目
  • 供应商树（vnd）主要表示公开发布的项目
    是一种概念上的东西,类似与git的分支,如果正常开发就按照 x,prs,vnd 的描述来填写即可。

API_SUBTYPE

在请求header中需要用到他

API_PREFIX

地址的前缀,如果不需要请填写 ‘/‘

API_VERSION

接口的版本，填写后是默认访问的版本

API_NAME

接口的名称，用于生成api文档，其他地方不使用

API_CONDITIONAL_REQUEST

条件请求默认为开启状态，这有利于客户端的缓存机制在可能的情况下缓存 API 请求。

API_STRICT

强制每次请求必须带版本，既

API_DEBUG

是否开启调试，开启后访问api会看到

API_DEFAULT_FORMAT

返回的类型，一般都是json
